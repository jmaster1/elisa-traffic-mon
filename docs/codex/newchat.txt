Работаем с `elisa-traffic-mon` как client app поверх `jmaster-web`.

Правила:
- отвечай по-русски;
- не запускай сборку, компиляцию, тесты, dev-server, браузер, Maven/Gradle lifecycle без прямой просьбы;
- если пользователь задает вопрос, нужен ответ, а не правки кода;
- если пользователь просит правку, сначала посмотри `git status --short` в `elisa-traffic-mon`;
- для поиска используй `rg` / `rg --files`, для ручных правок - `apply_patch`;
- не создавай миграции/changelog, новые зависимости, крупные рефакторинги или массовое форматирование без прямой просьбы.

Контекст по необходимости:
- для общей ориентации прочитай `elisa-traffic-mon/docs/codex/prompt.txt`;
- если нужна карта workspace/framework boundary, прочитай `jmaster-web/docs/codex/workspace-map.md`;
- если задача уходит в framework, прочитай `jmaster-web/docs/codex/workspace-prompt.txt` и `jmaster-web/description.txt`;
- основные зоны:
  - `elisa-traffic-mon/src/main/java/jmaster/etm/server`
  - `elisa-traffic-mon/src/main/resources/templates`
  - `elisa-traffic-mon/src/main/resources/static`
  - `elisa-traffic-mon/src/main/resources/db`

Framework boundary:
- сначала ищи локальное решение в `elisa-traffic-mon`;
- иди в `jmaster-web`, только если поведение общее, класс из `jmaster.*`, или локальная правка дублирует framework-логику.
